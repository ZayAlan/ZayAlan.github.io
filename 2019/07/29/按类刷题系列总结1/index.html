1155. Heap Paths (30) [深搜回溯，堆]

最后自己AC了，但是花了一个半小时的时间，原因是DFS不够熟练，不断地debug才修成正确的。
（忽然发现是误打误撞做对的，在判断大小根堆的部分中我写的代码是错误的，只是非常巧合的OJ上的测试案例刚好能对上。。。）
我的（修改后正确的）思路：

  1. DFS先序遍历给的堆，每条完整的路径存入vector中。
  2. 设置sum1，sum2来作为判断大小根堆的依据。
  3. 倒序访问vector中的每条路径并打印出来，同时分别记录比较后大于等于、小于等于的次数和元素的个数。
  4. 比较sum1、sum2与元素个数的大小，判断是哪种堆。
  5. DFS的写法是根据完全二叉树的特性写的。

1151. LCA in a Binary Tree (30) [树的遍历，中序前序建树，LCA算法]

在做这题的时候，我原本的想法是用数组根据先序和中序序列建树处理，但是题目中所给的元素并没有明确的规定，所以使用数组进行建树的结点对应问题无法解决。（同时自己对建树也不熟练）
最后自己还是没写出来，参考柳神的题解，她的做法非常屌，不需要建树（但是我还是要学一下怎么建树）：

  1. 设置了一个map，用来给每个inorder序列中的元素设置其对应的inorder序列的位置。好处是在后续判断树的结点中是否含有题给元素时可以直接在map中查询判断。（此法可以在其他题目中学以致用）

  2. 已知某个树的根结点，若a和b在根结点的左边，则a和b的近公共祖先在当前⼦子树根结点的左⼦子树寻找，如果a和b在当前⼦子树根结点的两边，在当前⼦子树的根结点就是a和b的近公共祖先，如果a和b在当前⼦子树根结点的右边，则a和b的近公共祖先就在当前⼦子树的右⼦子树寻找。

  3. 还有一个新学的小技巧，使用vector，可以在获取到元素数目后，使用resize(n)为其设置大小操作。（此法可以在其他题目中学以致用）

1147. Heaps (30) [堆，树的遍历]

涉及到完全二叉树的层次遍历的性质，注意其层次遍历的结点中含有子节点的结点个数为n/2个，所以本题在判断是否是堆的时候知道这个性质后能够极大地简化代码。
另外，大根堆还是小根堆可以通过层次序列的前两个数的比较就能做出初步的判断。
