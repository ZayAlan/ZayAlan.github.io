Android学习之路--活动的启动模式

发表于 2018-05-14

在实际项目中我们应该根据特定的需求为每个活动指定恰当的启动模式，一共有四种。

启动模式 作用 示意图
standard 活动默认的启动模式，每当启动一个新活动，它就会在返回栈中入栈，并处于栈顶的位置，每次启动一个活动都会创建该活动的一个新的实例
singleTop 在启动活动时如果发现返回栈的栈顶已经是该活动，则认为可以直接使用它，不会再创建新的活动实例，如果不是该活动，则会继续创建新的实例
singleTask 可以很好的解决重复创建栈顶活动的问题，每次启动该活动时系统首先会在返回栈中检查是否存在该活动的实例，如果发现已存在则直接使用该实例，并把这个活动之上的所有活动统统出栈，如果没有发现就会创建一个新的活动实例
singleInstance 启用一个新的返回栈来管理这个活动，解决共享活动实例的问题

发表于 2018-05-13 | 更新于 2018-05-14

Intent 是Android程序中各组件之间进行交互的一种方式，不仅可以指明当前组件想要执行的动作，还可以在不同组件之间传递数据。一般可被用于启动活动、启动服务以及发送广播等场景。

下面建立在已有两个Activity（FirstActivity和SecondActivity）的基础上进行。

使用显示Intent

启动活动，Intent的“意图”明显，故称之为显示Intent。通过修改FirstActivity中按钮的点击事件:

  • 用Intent的构造函数Intent(Context packageContext, Class<?>cls)重载，第一个参数Context要求提供一个启动活动的上下文，第二个参数Class则是指定想要启动的目标活动。这个构造函数构建出了Intent的“意图”
  • Activity类中提供了一个startActivity()方法，它接收一个Intent参数用于启动活动。

运行程序，点击button1，就可以启动SecondActivity这个活动了。

使用隐式Intent

并不明确指出想要启动哪一个活动，而是通过指定一系列抽象的action和category等信息，然后交由系统去分析这个Intent，找出合适的活动去启动。这种方式称之为隐式Intent

  • 首先在AndroidManifest.xml中配置SecondActivity的<activity>标签下的<intent-filter>内容，指定SecondActivity能够响应的action和category

    只有action和category中的内容同时能够匹配上Intent中指定的两项时，这个活动才能响应该Intent

可在<intent-filter>标签中再配置一个<data>标签，用于更精确地指定当前活动能够响应什么类型的数据，具体可配置情况如下

<data> 作用
android:scheme 用于指定数据的协议部分，如上例的https部分
android:host 用于指定数据的主机名部分，如上例的 www.baidu.com 部分
android:port 用于指定数据的端口部分，一般紧随在主机名之后

Android学习之路--Log工具

发表于 2018-05-13

日志工具类Log（android.util.Log）:

  1. Log.v() 打印意义最小的日志信息，对应verbose，是日志里面级别最低的一种。
  2. Log.d() 打印调试信息，对应debug，比verbose级别高。
  3. Log.i() 打印比较重要的数据，可以帮助分析用户行为数据，对应info，比debug级别高。
  4. Log.w() 打印警告信息，提示程序在这个地方可能会有潜在的风险，对应warn，比info级别高。
  5. Log.e() 用于打印错误信息，对应error，比warn级别高。

eg:

tag参数（“HelloWorldActivity”）：一般传入当前的类名，主要用于对打印信息进行过滤。

msg参数（“onCreate execute”）：即想要打印的具体的内容。

tips：

  1. logd，logi，logv，logw，loge等按下tab自动补全
  2. 在onCreate()外输入logt后按下tab，会以当前类名作为值自动生成一个TAG常量
  3. logcat添加过滤器
  4. 关键字过滤，支持正则表达式
